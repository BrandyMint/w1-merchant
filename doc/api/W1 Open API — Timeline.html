Получение событий пользователя

GET /OpenApi/timeline/?maxItems={maxItems}&date={date}

Возвращает maxItems событий пользователя, отсортированных по дате в обратном порядке, до даты date не включительно. Здесь и далее под датой понимается дата-время. В результат дополнительно включаются все события, у которых дата совпадает с датой самого раннего вернувшегося события (SELECT TOP (maxItems) WITH TIES … ORDER BY EventDate DESC).

Сценарий использования:

  1. Делаем запрос с параметрами maxItems = 50 и date = текущая дата и время.
  2. Если вернулось менее 50 событий, то мы получили все события пользователя и выполнять больше запросов не требуется. ВЫХОД.
  3. Если пользователю нужно получить более ранние события, то делаем запрос с датой самого раннего пришедшего события.
  4. Переход на шаг 2.

Пример запроса:

GET /OpenApi/timeline/?maxItems=50&date=2114-05-23T23%3A59%3A59.999Z
