Получение информации о текущей сессии

GET /OpenApi/sessions/current

Данный метод позволяет получить информацию только о текущей используемой сессии.

В случае успеха возвращает HTTP-статус 200 Ok и информация о текущей сессии в теле ответа.

Подробно о назначении и формате поля Scope можно ознакомиться в документации по OAuth.

Пример тела ответа:

{

   // Идентификатор приложения OpenApi

   "ClientId":"openapi_client",

   // Дата создания токена

   "CreateDate":"2012-11-26T06:59:17.863Z",

   // Дата/время истечения действия токена

   "ExpireDate":"2013-01-24T10:47:18.587Z",

   // Операции доступные по данному токену

   "Scope":"GetBalance",

   // Время, на которое продлевается действие токена в секундах

   "Timeout":1800,

   // Токен сессии

   "Token":"EA780BBD-6C1A-4032-9740-F740A9DA0420",

   // Номер кошелька

   "UserId":100000000000,

   // Токен дочерней сессии

   "PrincipalSessionKey":"EA680BBD-6C1A-4032-9740-F740A9DA0420"

}

Отправка одноразового пароля

POST /OpenApi/password/otp

Существуют ограничения на количество одноразовых паролей для одного пользователя. В час можно отправить не более 5 одноразовых паролей, в сутки - не более 15.

Время действия одноразового пароля ограничено максимум 60 минутами.

Пример запроса:

{

   // Email или телефон пользователя, куда будет отправлен пароль

   "Login":"79121111111",

   // Не обязательный параметр, время жизни пароля в минутах

   "ExpireInMinutes":13

}

Восстановление пароля

POST /OpenApi/password/?sendto={login}

Генерирует новый пароль пользователя и отправляет его на  указанный телефон или e-mail. Новый пароль действует в течение 30 минут и становится постоянным после первого входа с его использованием.

Создание сессии по логину и паролю

POST /OpenApi/sessions

Подробно о назначении и формате поля Scope можно ознакомиться в документации по OAuth.

Пример тела запроса:

{

   // Номер кошелька, email или номер телефона

   // пользователя от имени, которого будет выдан токен

   "Login":"79111111111",

   // Пароль пользователя

   "Password":"123",

   // Разрешения, которые будут предоставлены

   "Scope":"All",

   // Данные о клиенте

   // (ключ/значение, список строго не регламентирован)

   "Params":[

      // IP-адрес удаленного клиента. Передается

      // обязательно, если приложение, использующее Open API,

      // запускается на сервере

      { "Name":"RemoteIp",   "Value":"122.33.44.55" },

      { "Name":"UserAgent",  "Value":"Mozilla/20" },

      { "Name":"Resolution", "Value":"1280x800" },

      ... // другие данные

   ]

}

   "ExpireDate":"2013-01-24T10:47:18.587Z",

   // Операции доступные по данному токену

   "Scope":"All",

   // Время, на которое продлевается действие токена в секундах

   "Timeout":1800,

   // Токен сессии

   "Token":"EA780BBD-6C1A-4032-9740-F740A9DA0420",

   // Номер кошелька

   "UserId":100000000000,

   // Токен дочерней сессии

   "PrincipalSessionKey":"EA680BBD-6C1A-4032-9740-F740A9DA0420"

}

Смена пароля

PUT /OpenApi/payments?providerId=ChangePassword

Позволяет сменить пароль пользователя, для которого получена сессия.

Пример тела запроса:

{

   // Название формы

   "FormId":"Index",

   // Набор параметров

   "Params":[

      { "FieldId":"OldPassword", "Value":"{хеш текущего пароля}" },

      { "FieldId":"NewPassword", "Value":"{хеш нового пароля}" }

   ]

}

Пример тела ответа:

{

  // Идентифкатор операции

  "PaymentId":10000000,

  // Тип операции

  "ProviderId":"ChangePassword",

  // Состояние

  "State": { "StateId":"Created", "Description":"" },

  // Шаг операции

  "Step":2,

  // Количество шагов операции

  "StepCount":3,

  // Данные формы для заполнения

  "Form": { ... }

   ]

}

Если в качестве двухфакторной аутентификации используется аутентификация по SMS, то отправляем код подтверждения:

POST /OpenApi/payments/{paymentId}/code

Пример тела ответа:

{

   // Тип получателя

   "RecipientType":"PhoneNumber",

   // Часть номера получателя

   "Recipient":"*******0000"

}

Подтверждаем изменение пароля:

PUT /OpenApi/payments/{paymentId}

Пример тела запроса:

{

   // Название формы

   "FormId":"$Final",

   // Набор параметров. Указываются в случае использования двухфакторной аутентификации

   "Params":[

      { "FieldId":"$OtpCode", "Value":"{код из SMS или TimeBased}" },

       // ИД подключенной схемы подтверждения операций необязателен для подтверждения по SMS

      { "FieldId":"$UsedUserOtpSchemeId", "Value":"{ИД схемы подтверждения операций}" }

   ]

}

Пример тела ответа:

{

  // Идентифкатор операции

  "PaymentId":10000000,

  // Тип операции

  "ProviderId":"ChangePassword",

  // Состояние

  "State": { "StateId":"Paid", "Description":"" },

  // Шаг операции

  "Step":3,

  // Количество шагов операции

  "StepCount":3,

  // Данные формы для заполнения

  "Form": { ... }

   ]

}

Регистрация пользователя

POST /OpenApi/registration/users

Пример тела запроса:

{

   // Адрес электронной почты или номер телефона

   "Login":"user@server.com",

   // Стандартизированное обозначение культуры пользователя

   "CultureId":"ru-RU",

   // Отправлять ли пароль на указанный телефон/e-mail

   "SendPassword":true,

   // Необязательно: отправлять ли код подтверждения на

   // регистрируемый адрес e-mail или телефон (по умолчанию true)

   "SendVerificationCode":true,

   // Необязательно: Personal (физ.лицо) / Business (юр.лицо)

   "AccountTypeId":"Personal",

   // Необязательно: код приглашения

   "Referrer":123456789012,

   // Необязательно: пароль пользователя

   "Password":null,

   // Необязательно: нужно ли открыть сессию сразу после регистрации

   "GenerateSession":false,

   // Необязательно: IP-адрес удаленного клиента. Передается

   // обязательно, если приложение, использующее Open API,

   // запускается на сервере

   "RemoteIp":"122.33.44.55",

   // Необязательно: данные для заполнения профиля пользователя

   "ProfileItems":[

      { "TypeId":"FirstName", "Value":"Ivan" },

      // ...

   ],

   // Необязательно: промо-код

   "PromoCodeId":"1127682",

   "CreateMerchant":false

}

Если указывается параметр GenerateSession:true, то параметр Password должен быть указан обязательно.

Если указывается параметр Password, то параметр SendPassword должен иметь значение false.

Пример ответа:

{

   // Полученный идентификатор пользователя

   "UserId":133333333333,

   // Описание открытой сессии, если GenerateSession=true

   "Session":null

}

Описания открытой сессии строится аналогично телу ответа в cоздании сесиии.

Провести запрос на идентификацию

POST /OpenApi/kyc/requests/{kycRequestId}

Отправляет запрос на идентификацию на модерацию

Единственный параметр - kycRequestId, идентификатор ранее созданного запроса.

В ответ возвращается структура KycRequest - запрос на идентификацию.

Получить список запросов на идентификацию

GET /OpenApi/kyc/requests?KycRequestStateId={KycRequestStateId}

Возвращает список запросов на идентификацию, созданных данным кошельком.

Доступен фильтр по состоянию запроса на идентификаию - KycRequestStateId.

В ответ возвращается набор структур KycRequest
