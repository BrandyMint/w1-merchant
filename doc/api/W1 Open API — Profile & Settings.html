Получение профиля пользователя

GET /OpenApi/profile

GET /OpenApi/profile?userId={userId}&types={types}

Необязательные параметры строки запроса:

  • userId - идентификатор пользователя;
  • types - типы атрибутов или группы типов атрибутов пользователя, перечисленные через запятую.

В случае, если пользователь не найден, возвращает 404 NotFound.

Пример ответа:

{

   "UserId":123456789012, // идентификатор пользователя

   "IsOnline":true,       // выполнен ли пользователем вход 

   "AccountTypeId":"Personal", // тип пользователя

   "MerchantStateId":"Active", //состояние единой кассы

   "UserAttributes":[     // атрибуты пользователя

      {

         "UserId":123456789012,

         "UserAttributeId":9999999,

         "UserAttributeTypeId":"Title",

         "VisibilityTypeId":"All",

         "IsReadOnly":false,

         "VerificationState":"Unverifiable",

         "Comment":"",

         "DisplayValue":"Mister X",

         "RawValue":""

      },

      ... // Список атрибутов пользователя

   ]

}

Сохранение профиля пользователя

POST /OpenApi/profile/updates

Регистрирует обновление профиля пользователя.

В зависимости от состава запрашиваемых изменений профиля, операция может требовать подтверждения SMS-кодом, высылаемым на телефон пользователя.

Тело запроса содержит список новых или обновленных значений пользовательских атрибутов, идентифицируемых их типами. Для удаления атрибута из профиля пользователя необходимо указать его в списке с пустым значением (“”).

Пример тела запроса:

{

   "Items":

   [

      {

         "TypeId" : "Title",

         "Value"  : "John"

      },

      {

         "TypeId" : "BirthDate",

         "Value"  : "1982-06-13"

      },

      ...

   ],

   "SendVerificationCode": false

}

Результатом обработки запроса является обновление профиля, которое может находиться в следующих состояниях:

  • Created - обновление зарегистрировано, для его применения необходимо подтверждение обновления с помощью SMS-кода;
  • Done - обновление применено.

Пример ответа:

{

   "ProfileUpdateId": 1212,

   "ProfileUpdateState": "Created"

}

Подтверждение обновления профиля пользователя

PUT /OpenApi/profile/updates/{profileUpdateId}

Ввод SMS-кода подтверждения для обновления профиля пользователя.

Пример тела запроса:

{

   "VerificationCode": "1234567890",

   "UserOtpSchemeId": 123

}

Пример ответа:

{

   "ProfileUpdateId": 1212,

   "ProfileUpdateState": "Done"

}

Возможные ошибки:

  • UNLOCK_CODE_NOT_MATCH - переданный код не соответствует коду подтверждения обновления профиля.

Повторная отправка кода подтверждения обновления профиля пользователя

POST /OpenApi/profile/updates/{profileUpdateId}/code

Тела запроса и ответа пустые.

Верификация атрибута пользователя

Начало верификации

POST /OpenApi/profile/attributes/{userAttributeId}/verification

Тело запроса пустое.

В случае, если атрибут пользователя не найден, возвращает 404 NotFound.

Пример ответа:

{

    // Идентификатор пользователя

    "UserId":100000000000,

    // Идентификатор атрибута пользователя

    "UserAttributeId":9999999,

    // Тип атрибута

    "UserAttributeTypeId":"PassIssuer",

    // Тип видимости атрибута

    "VisibilityTypeId":"None",

    // Флаг, указывающий, что значение атрибута

    // только для чтения

    "IsReadOnly":false,

    // Состояние верификации для данного атрибута

    "VerificationState":"Processing",

    // Комментарий

    "Comment":"",

    // Значение атрибута

    "DisplayValue":"Santa Claus",

    // Значение атрибута

    "RawValue":""

}

Завершение верификации

PUT /OpenApi/profile/attributes/{userAttributeId}/   

     verification?verificationCode={verificationCode}

Тело запроса пустое. В обязательном параметре строки запроса verificationCode передается код верификации атрибута пользователя. В случае, если атрибут пользователя не найден, возвращает 404 NotFound.

Пример ответа:

{

    // Идентификатор пользователя

    "UserId":100000000000,

    // Идентификатор атрибута пользователя

    "UserAttributeId":9999999,

    // Тип атрибута

    "UserAttributeTypeId":"PassIssuer",

    // Тип видимости атрибута

    "VisibilityTypeId":"None",

    // Флаг, указывающий, что значение атрибута

    // только для чтения

    "IsReadOnly":false,

    // Состояние верификации для данного атрибута

    "VerificationState":"Verified",

    // Комментарий

    "Comment":"",

    // Значение атрибута

    "DisplayValue":"Santa Claus",

    // Значение атрибута

    "RawValue":""

}

Пользовательские настройки приложения

Пользовательские настройки приложения позволяют приложению сохранить произвольные данные, связанные с пользователем, в виде набора пар “имя - значение” (без группировки), либо разделенные по группам, в виде набора троек “группа - имя - значение”. В каждой группе может быть только одна настройка с определенным именем.

Получение пользовательских настроек приложения

GET /OpenApi/settings

GET /OpenApi/settings?groupId={groupId}&name={name}

Необязательные параметры строки запроса:

  • groupId - фильтр по группе пользовательских настроек приложения;
  • name - фильтр по имени пользовательской настройки приложения.

Пример ответа:

{  

  "Items": [

     {

        "GroupId":"Appearance",

        "Name":"ColorScheme",

        "Value":"Modern"

     },

     ... // остальные настройки

  ]

}

Сохранение пользовательских настроек приложения

PUT /OpenApi/settings

Пример тела запроса:

{

  "Items": [

     {

        "GroupId":"Appearance",

        "Name":"ColorScheme",

        "Value":"Modern"

     },

     ... // остальные настройки

  ]

}

Тело ответа пустое.
