Создание перевода

POST /OpenApi/transfers

Сервис поддерживает два способа идентификации переводов: каждый перевод имеет уникальный идентификатор, назначенный сервисом (обозначается TransferId), а также может иметь уникальный идентификатор, назначенный пользователем (обозначается ExternalId).

В случае, когда пользователь назначает переводам идентификаторы, нельзя назначать один и тот же идентификатор различным переводам. Не рекомендуется использовать длинные (более 40 символов) строки в качестве идентификаторов переводов.

Пользователь может не назначать создаваемым переводам идентификаторы. В этом случае поле ExternalId не передается.

Пример тела запроса:

{

   // Получатель перевода — номер кошелька, телефона или e-mail

   "Recipient":"200000000000",

   // Сумма перевода

   "Amount":10.0,

   // Идентификатор валюты, согласно ISO 4217

   "CurrencyId":643,

   // Внешний уникальный идентификатор перевода.

   "ExternalId":"RS-259",

   // Описание перевода

   "Description":"Transfer description",

   // Дата/время истечения перевода

   "ExpireDate":"2012-04-15T09:25:15.400",

   // Код разблокировки перевода

   "UnlockCode":""

}

Если указан UnlockCode, то перевод зачисляется только после ввода кода получателем или отменяется после наступления ExpireDate. 

Если в ответ приходит ошибка BALANCE_HOLD_STATE_ID_CHANGE_ERROR, то это означает, что была попытка создания перевода по уже использованной блокировки средств.

В случае успеха возвращает HTTP-статус 201 Created и созданный перевод в теле ответа.

Пример тела ответа можно посмотреть в пункте Получение деталей перевода.

В случае ошибки возвращает 400 Bad Request и один из следующих кодов:

Код ошибки

Описание

LOGIN_FORMAT_ERROR

Не найден идентификатор получателя, либо значение Recipient имеет неподдерживаемый формат.

SELF_OPERATION_ERROR

Нельзя выставить счет самому себе.

AMOUNT_RANGE_ERROR

Сумма не попадает в допустимый диапазон.

CURRENCY_ID_NOT_FOUND

Неизвестная валюта платежа.

EXPIRE_DATE_RANGE_ERROR

Неверная ExpireDate.

OPERATION_BANNED

Для переданных кошельков проведение данной операции завершено. Свяжитесь со службой поддержки для выяснения подробностей.

BALANCE_IS_INSUFFICIENT

Недостаточно денег на счете для проведения данной операции.

ORDER_ID_ALREADY_USED

Повторное использование внешнего идентификатора перевода. Приложение должно получить детали перевода с этим внешним идентификатором и обновить его статус в своей учетной системе при необходимости.

BALANCE_HOLD_STATE_ID_CHANGE_ERROR

Смена состояния блокировки средств на требуемое невозможна.

Получение деталей перевода

GET /OpenApi/transfers/{transferId}

GET /OpenApi/transfers/?externalId={externalId}

Получить детали перевода можно как по идентификатору, назначенному сервисом, так и по идентификатору, назначенному пользователем (если такой имеется).

Пример ответа:

{

   // Идентификатор перевода

   "TransferId":444444444444,

   // Идентификатор получателя или отправителя перевода

   "UserId":200000000000,

   // Имя отправителя или получателя перевода

   "UserTitle":"200000000000@W1",

   // Направление перевода:

   // Inc - входящий перевод

   // Out - исходящий перевод

   "Direction":"Out",

   // Сумма перевода

   "Amount":10.0000,

   // Сумма комиссии перевода

   "CommissionAmount":0.0000,

   // Идентификатор валюты, согласно ISO 4217

   "CurrencyId":643,

   // Внешний идентификатор перевода, передается только отправителю перевода.

   "ExternalId":"RS-259",

   // Описание перевода

   "Description":"Transfer description",

   // Дата создания перевода

   "CreateDate":"2012-04-13T09:26:45.600Z",

   // Дата последнего изменения состояния перевода

   "UpdateDate":"2012-04-13T09:26:45.600Z",

   // Дата истечения перевода

   "ExpireDate":"2012-04-15T09:25:15.400Z",

   // Состояние перевода

   "TransferStateId":"Accepted",

   // Флаг, указывающий на наличие параметра UnlockCode у перевода

   "IsProtected":true,

   // Код разблокировки перевода, передается только отправителю перевода.

   "UnlockCode":""

}

Состояние перевода может принимать следующие значения:

  • Accepted – успешный перевод, деньги зачислены получателю;
  • Canceled – перевод отменен отправителем;
  • Rejected – перевод отвергнут получателем;
  • Expired – истек срок для подтверждения получения, деньги возвращены отправителю;
  • Processing – перевод находится в обработке системой;
  • Created – получение не подтверждено получателем.

Изменение состояния перевода

POST /OpenApi/transfers/{transferId}/state

Пример тела запроса:

{

   "StateId":"Accepted",

   "UnlockCode":"test"

}

В ответе возвращается структура Transfer, аналогично методу получения деталей перевода.

Получение истории переводов

GET /OpenApi/transfers/?

pageNumber={pageNumber}&itemsPerPage={itemsPerPage}

&fromCreateDate={fromCreateDate}&toCreateDate={toCreateDate} &operationType={operationType}

&stateFilter={stateFilter}&currencyId={currencyId}

&searchString={searchString}&direction={direction}

Необязательные параметры строки запроса:

  • pageNumber — номер страницы, начиная с 1
  • itemsPerPage — количество элементов на странице
  • fromCreateDate — начало периода фильтрации переводов
  • toCreateDate — конец периода фильтрации переводов
  • operationType — фильтр по типу операции
  • stateFilter — фильтр по состояниям выбираемых переводов, кроме значений статусов, описанных в пункте получение деталей перевода, можно использовать группы статусов:
  • “Waiting” - в одном из статусов “Created” или “Processing”;
  • “Aborted” - в одном из статусов “Canceled” или “Expired” или “Rejected”.
  • currencyId — фильтр по валюте выбираемых переводов
  • searchString — фильтр для поиска по описанию перевода, или по имени пользователя, который являлся стороной перевода
  • direction — фильтр по направлению перевода:
  • Inc - входящий;
  • Out - исходящий.

Структура тела ответа:

  • TotalCount – общее количество переводов, удовлетворяющих фильтрам
  • Items – запрошенная страница, элементы которой описывают переводы (если передан параметр itemsPerPage, содержит не более itemsPerPage элементов). Переводы упорядочены по времени создания, от подзних к ранним.
