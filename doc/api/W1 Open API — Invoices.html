Создание счета

POST /OpenApi/invoices

Сервис поддерживает два способа идентификации счетов: каждый счет имеет уникальный идентификатор, назначенный сервисом (обозначается InvoiceId), а также может иметь уникальный идентификатор, назначенный пользователем (обозначается OrderId).

В случае, когда пользователь назначает счетам идентификаторы, нельзя назначать один и тот же идентификатор различным счетам. Не рекомендуется использовать длинные (более 40 символов) строки в качестве идентификаторов счетов.

Пользователь может не назначать создаваемым счетам идентификаторы. В этом случае поле OrderId не передается.

Пример тела запроса:

{

   // Идентификатор валюты, согласно ISO 4217

   "CurrencyId":643,

   // Сумма счета

   "Amount":1.0000,

   // Описание счета

   "Description":"test invoice",

   // Дата/время истечения счета

   "ExpireDate":null,

   // Внешний уникальный идентификатор счета

   "OrderId":null,

   // Получатель счета — номер кошелька, телефона или e-mail

   "Recipient":"100000000002"

}

В случае успеха возвращает HTTP-статус 201 Created и созданный счет в теле ответа.

Пример тела ответа можно посмотреть в пункте Получение деталей счета.

Получение деталей счета

GET /OpenApi/invoices/{invoiceId}

GET /OpenApi/invoices/?orderId={orderId}

Пример ответа:

{

   // Идентификатор счета, назначенный сервисом

   "InvoiceId":342342342342,

   // Идентификатор отправителя счета

   "FromUserId":100000000001,

   // Идентификатор получателя счета

   "ToUserId":100000000002,

   // Идентификатор контрагента по счету

   "UserId":100000000002,

   // Имя контрагента по счету

   "UserTitle":"100000000002@W1",

   // Направление счета:

   // Inc - входящий счет

   // Out - исходящий счет

   "Direction":"Out",

   // Сумма счета

   "Amount":1.0000,

   // Идентификатор валюты, согласно ISO 4217

   "CurrencyId":643,

   // Внешний идентификатор счета, передается только отправителю

   "OrderId":342342342342,

   // Описание счета

   "Description":"test invoice",

   // Дата создания счета

   "CreateDate":"2012-04-13T10:54:18.657Z",

   // Дата последнего изменения счета

   "UpdateDate":"2012-04-13T10:54:18.657Z",

   // Дата истечения счета

   "ExpireDate":null,

   // Состояние счета

   "InvoiceStateId":"Created",

   // Уплаченная сумма по счету

   "PaidAmount":0.0000,

   // Пользовательский комментарий к счету

   // (доступен только написавшему его пользователю)

   "Comment":null,

   // Адрес для перехода на сайт магазина в случае успешной оплаты

   "SuccessUrl":"http://store.com/success",

   // Теги счета

   "Tags":null

}

Состояние счета может принимать следующие значения:

  • Accepted – оплачен;
  • Canceled – отменен отправителем;
  • Created – счет создан, но не получен;
  • Expired – истек срок действия счета;
  • Processing – счет находится в процессе подтверждения;
  • Received – счет получен, ожидает принятия;.
  • Rejected – счет отвергнут получателем.

Получение списка счетов

GET /OpenApi/invoices?

pageNumber={pageNumber}&itemsPerPage={itemsPerPage}

&invoiceStateId={invoiceStateId}&direction={direction}

&fromCreateDate={fromCreateDate}&toCreateDate={toCreateDate}

&searchString={searchString}

Обязательные параметры строки запроса:

  • pageNumber — номер страницы, начиная с 1
  • itemsPerPage — количество элементов на странице

Необязательные параметры строки запроса:

  • invoiceStateId — фильтр по состоянию счета.
  • direction — фильтр по направлению счета:
  • Inc - входящий;
  • Out - исходящий.
  • fromCreateDate - фильтр по дате создания счета.
  • toCreateDate - фильтр по дате создания счета.
  • searchString - фильтр по строке поиска.

Пример ответа:

{

   "Invoices":[

     {

       "InvoiceId":342342342342,

       "UserId":100000000000,

       "UserTitle":"100000000000@W1",

       "Direction":"Out",

       "Amount":1.0000,

       "CurrencyId":643,

       "OrderId":342342342342,

       "Description":"test invoice",

       "CreateDate":"2012-04-13T10:54:18.657Z",

       "UpdateDate":"2012-04-13T10:54:18.657Z",

       "ExpireDate":null,

       "InvoiceStateId":"Created",

       "PaidAmount":0.0000,

       "Comment":null,

       "SuccessUrl":"http://store.com/success",

       "Tags":null

     },

     ... // остальные счета

   ],

   // Общее число счетов,

   // удовлетворяющих заданным критериям

   "TotalCount":27 

}

Изменение состояния счета

POST /OpenApi/invoices/{invoiceId}/state

Пример тела запроса:

{

   "InvoiceStateId":"Accepted"

}

Пример тела ответа можно посмотреть в пункте Получение деталей счета.

Установка пользовательского комментария к счету

PUT /OpenApi/invoices/{invoiceId}/comment

Пример тела запроса:

{

   "Comment":"Commentary is very useful",

   "Tags":"Shop"

}

Пример тела ответа можно посмотреть в пункте Получение деталей счета.

Зачисление со счета невыясненных платежей на основной счет интернет-магазина

POST /OpenApi/invoices/{invoiceId}/operations/accept

Пример тела ответа можно посмотреть в пункте Получение информации о платежах по счету.

Возврат со счета невыясненных платежей

POST /OpenApi/invoices/{invoiceId}/operations/refund

Пример тела запроса:

{

   "RefundUserId": 123456789012

}

Пример тела ответа можно посмотреть в пункте Получение информации о платежах по счету.

Привязать счет к кошельку

Привязывает счет к кошельку сессии.

POST /OpenApi/invoices/{invoiceId}/bind

В случае успеха возвращает HTTP-статус 200.
