Регистрация партнера

POST /OpenApi/partner/activate

Отправляется пустой POST-запрос, если метод отработал корректно, вернется пустая строка, в противном случае возникнет ошибка.

При успешном ответе партнер имеет статус “Ожидает модерации”, после проверки менеджерами статус переходит в Active либо в Closed

Создание промо-кода

POST /OpenApi/partner/promoCodes

Пример запроса:

{

   // Является ли промо-код одноразовым. Необязательный параметр.

   "IsOneTime":true,

   // Дата, до которой действует промо-код. Необязательный параметр.

   "ExpireDate":"2015-01-01T00:00:00",

   // Действие промо-кода в днях. Необязательный параметр.

   "TariffDuration": 10,

   // количество создаваемых промокодов, от 1 до 10. Необязательный параметр, по умолчанию 1.

   "Count": 10,

   "TariffUserId": 123456789012,

}

Примечание:

Хотя бы один из параметров IsOneTime, ExpireDate должен быть задан

Получение данных по партнерской программе

GET /OpenApi/partner/statistics?dateFrom={dateFrom}

&dateTo={dateTo}&currencyId={currencyId}

Возвращает данные по партнерской программе, {currencyId} - необязательный фильтр по валюте.

Пример ответа:

{

  [

    {

      // Кошелек привлеченного клиента

      "ReferralUserId":123456789012,

      // Название

      "Title":"Test",

      // Url магазина

      "MerchantUrl":"mysupersite.com",

      // Код валюты

      "CurrencyId": 643,

      // Оборот

      "Turnover": 1231.0000,

      // Выплаченный партнерский бонус

      "BonusPaid": 12.3100,

      // Статус: Active, Inactive

      "State":"Active",

       // промокод, по которому зарегистрировался магазин

      "PromoCode":"hkU93khs1"

    },

    ...

  ]

}

Получения статуса партнера

GET /OpenApi/partner/state

Возвращает статус партнера, Active, Pending или Closed

Пример ответа:

"Active"

Получение списка промокодов

GET /OpenApi/partner/promoCodes?onlyLast={onlyLast}

Возвращает список сгенерированных промокодов, {onlylast} - необязательный параметр, если указан, то возвращается только последний сгенерированный код.

Пример ответа:

{

  [

    {

      "PromoCodeId":"axNcl1nmi"

      "IsOneTime":"true"

    },

    {

      "PromoCodeId":"axNcl1nmi"

      "IsOneTime":"false"

    },

    ...

  ]

}

Получение данных по партнерской программе с разбиением по дням

GET /OpenApi/partner/statistics/days?dateFrom={dateFrom}&dateTo={dateTo}&currencyId={currencyId}&merchantUserId={merchantUserId}

Параметры merchantUserId и currencyId являются необязательными.

Пример ответа:

{

  [

    {

      "BonusDate":"2014-06-20T00:00:00",

      "BonusAmount":"150.23",
      "CurrencyId":"643"   
   },
    {

      "BonusDate":"2014-06-21T00:00:00",

      "BonusAmount":"10.19",
      "CurrencyId":"643"   
   }
   ...

  ]

}

Получение информации о партнере

GET /OpenApi/partner/profile

Ответ:

{

   "PartnerState": "Active", //состояние, Active, Pending, None

   "FeeScheme": "Regressive", //идентификатор схемы вознаграждения

   //описание схемы партнерского вознаграждения

   "PartnerBonus": [

      {

         "BonusPercent": 15,

         "DurationDayCount": 180

      },

      {

         "BonusPercent": 10,

         "DurationDayCount": 360

      },

      {

         "BonusPercent": 5

      }

   ],

   //доступные тарифы для использования в промокодах

   "AvailableTariff": [

      150950864957,

      169318482526

   ]

}

Описание схемы партнерского вознаграждения строится следующим образом:
BonusPercent - процент, получаемый партнером от общего вознаграждения W1 на протяжении DurationDayCount дней. Если записей несколько, тогда записи сортируются по возрастанию значения DurationDayCount (если значения нет, то запись ставится в конец списка), и производится поиск записи с наибольшим значением DurationDayCount, не превышающим количество дней, пройденных со дня регистрации магазина до момента начисления вознаграждения.

В вашеуказанном примере вычисление производится следующим образом: если прошло 50 дней, то будет действовать запись с 15%, если 180 - то также 15%, если 200 дней, тогда 10, если 2 года, тогда 5 %.
